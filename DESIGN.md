## Visual Thesis

**"Technical warmth on a dark stage -- collaboration performs in light."**

The board is a dark canvas where every user action creates color and movement. Cards rest in structured columns. Cursors glow with user-specific colors. Presence is the brightest thing on screen. The aesthetic sits between Linear's sleekness and Figma's creative energy: precise, well-crafted, but alive.

**Dark-only. No light mode.** Every PM tool defaults to white. FlowBoard uses darkness so colored cursors, avatars, and presence indicators pop. This is a design position, not a toggle.

---

## Aesthetic Direction

**Technical Warmth** -- grid-disciplined layout, clean card rectangles, defined column containers. Not cold/clinical like Linear. Not corporate like Trello. Not playful like Notion. Warm because of the vivid cursor colors and subtle glow effects against the dark canvas.

**Decoration level: Intentional** -- cursor glow effects, subtle card elevation on hover/drag, presence animations. No gratuitous texture, grain, or decorative elements. Light itself is the decoration.

## Layout

### Board View (primary layout)

Grid-disciplined. Strict column grid, horizontal scroll for overflow.

```
+--[ Board Header ]-------------------------------------------+
| FlowBoard logo    Board Name            [Avatar] [Avatar]   |
+-------------------------------------------------------------+
|                                                              |
| +--Column--+ +--Column--+ +--Column--+ +--Column--+ [+ Add] |
| | Header   | | Header   | | Header   | | Header   |         |
| |----------| |----------| |----------| |----------|         |
| | [Card]   | | [Card]   | | [Card]   | | [Card]   |         |
| | [Card]   | | [Card]   | |          | | [Card]   |         |
| | [Card]   | |          | |          | |          |         |
| | [+ Add]  | | [+ Add]  | | [+ Add]  | | [+ Add]  |         |
```

- Column container: `--bg-surface` background, `--border-subtle` border, `--radius-lg` corners
- Cards: `--bg-card` background, `--border-subtle` border, `--radius-md` corners

## Motion System

### Philosophy

**"Inertia and settling. Nothing teleports. Everything has mass."**

Motion is not decoration in FlowBoard -- it's the core value proposition. Cursor movement, card dragging, presence transitions: these ARE what the recruiter watches.

### Motion Tokens

```css
:root {
  --duration-instant: 100ms;
  --duration-fast: 200ms;
  --duration-normal: 300ms;
  --duration-slow: 400ms; /* Maximum. Nothing over 400ms. */

  --ease-out: cubic-bezier(0.16, 1, 0.3, 1);
  --ease-in-out: cubic-bezier(0.65, 0, 0.35, 1);
  --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);
}
```

### Animation Specifications

| Action | Duration | Easing | Details |
|--------|----------|--------|---------|
| Card drag follow | Continuous | `spring(1, 80, 12)` | 40ms lag behind cursor. +/-2deg rotation based on drag velocity |
| Card drop settle | 350ms | `spring(1, 120, 14)` | Single bounce into position |
| Card created | 250ms | `ease-out` | Fade in from 0 + scale from 0.95 |
| Card archived | 300ms | `ease-in` | Fade out + scale to 0.95 |
| Remote cursor move | Continuous | Linear interpolation | 60fps with 60ms lerp smoothing |
| Cursor idle breathe | 3s cycle | Sine | Glow radius oscillates 8px to 14px |
| Presence enter | 400ms | `ease-out` | Cursor fades in from screen edge, slides to position |
| Presence leave | 400ms + 600ms glow | `ease-in` | Cursor fades to 0. Glow lingers 600ms then dissolves (ghost trace) |
| Column card cascade | 200ms + 30ms stagger | `ease-in-out` | When a card is added/removed, other cards shift with stagger |
| Modal open | 250ms | `ease-spring` | Scale from 0.95 + fade in |
| Modal close | 200ms | `ease-in` | Scale to 0.97 + fade out |
| Tooltip appear | 150ms | `ease-out` | Fade + 4px translate-y |
| Button hover | 100ms | `ease-out` | Background color shift |

### Framer Motion Config (React)

```typescript
// Card drag
const dragTransition = {
  type: "spring",
  stiffness: 80,
  damping: 12,
  mass: 1,
};

// Card drop
const dropTransition = {
  type: "spring",
  stiffness: 120,
  damping: 14,
  mass: 1,
};

// Layout animations (cards shifting in columns)
const layoutTransition = {
  type: "spring",
  stiffness: 200,
  damping: 25,
  mass: 0.8,
};
```

## Component Patterns

### Card (at rest)

```
+----------------------------------------------+
| [Label] [Label]                               |
| Card title text goes here                     |
| 12px muted text: due date, assignee count     |
| [Avatar -8px overlap] [Avatar]     [Due: Mon] |
+----------------------------------------------+
```

- Background: `--bg-card`
- Border: 1px solid `--border-subtle`
- Border radius: `--radius-md`
- Padding: 12px 16px
- Shadow: `--shadow-card`
- Title: DM Sans 500, `--text-md`, `--text-primary`
- Meta: DM Sans 400, `--text-xs`, `--text-muted`

### Card (hover)

- Background: `--bg-card-hover`
- Shadow: `--shadow-card-hover`
- Cursor: grab

### Card (dragging)

- Background: `--bg-card-active`
- Shadow: `--shadow-card-drag`
- Border: 1px solid `var(--user-color)` (dragging user's color)
- Rotation: +/-2deg based on velocity
- Scale: 1.02
- Cursor: grabbing
- Opacity: 0.95
- z-index: 1000

### Card (being edited by another user)

- Border-left: 3px solid `var(--user-color)`
- Subtle glow: `box-shadow: 0 0 0 1px var(--user-color), 0 0 8px rgba(var(--user-color-rgb), 0.15)`
- "Maria is editing..." indicator below card in `--text-muted` + user color dot

### Column Container

- Background: `--bg-surface`
- Border: 1px solid `--border-subtle`
- Border radius: `--radius-lg`
- Width: 280px
- Padding: 12px
- Header: Space Grotesk 600, `--text-lg`, `--text-primary`
- Card count badge: JetBrains Mono 400, `--text-xs`, `--text-muted`

### Avatar

- Size: 28px (board header), 20px (card assignees)
- Border: 2px solid `--bg-base` (creates separation in groups)
- Border radius: `--radius-full`
- Online indicator: 8px circle, `--accent-success`, positioned bottom-right
- Group overlap: -8px margin-left

### Cursor (remote user)

```
     ^
    / \        <- Filled triangle in user color
   /   \
  +-----+
  | Name |     <- Pill label, user color background, white text
  +------+
```

- Cursor size: 16x20px triangle
- Name pill: DM Sans 400, `--text-xs`, white text on user-color background
- Glow: `filter: drop-shadow(0 0 8px var(--user-color))`
- Idle animation: glow breathe (3s cycle)
- Leave animation: fade out 400ms, glow persists 600ms additional

### Button

- Primary: `--accent` background, `#0C1017` text, `--radius-md`
- Secondary: transparent, `--border-subtle` border, `--text-primary` text
- Ghost: transparent, no border, `--text-muted` text, hover: `--bg-card`
- Height: 32px (sm), 36px (md), 40px (lg)
- Font: DM Sans 500, `--text-sm`

### Input

- Background: `--bg-card`
- Border: 1px solid `--border-subtle`
- Focus border: `--accent`
- Text: `--text-primary`
- Placeholder: `--text-muted`
- Border radius: `--radius-md`
- Padding: 8px 12px

### Toast / Notification

- Background: `--bg-elevated`
- Border: 1px solid `--border-subtle`
- Border-left: 3px solid (accent color based on type)
- Shadow: `--shadow-modal`
- Duration: 4s auto-dismiss
- Entry: slide from right, 300ms ease-out
- Exit: fade + slide right, 200ms ease-in

## Anti-Patterns (do NOT use)

- Purple/violet gradients as accent
- 3-column icon grids for feature marketing
- Centered everything with uniform spacing
- Uniform bubbly border-radius on all elements
- Gradient buttons as primary CTA
- Glassmorphism / backdrop-filter blur on cards (performance killer with 17+ cards + animations)
- Pure white (#FFFFFF) text on dark backgrounds (use #E2E8F0)
- Pure black (#000000) backgrounds (use #0C1017)

## Accessibility Notes

- All text meets WCAG AA contrast on dark backgrounds (verified):
  - `--text-primary` (#E2E8F0) on `--bg-card` (#1A2233): ratio 9.8:1
  - `--text-muted` (#64748B) on `--bg-card` (#1A2233): ratio 3.2:1 (use only for non-essential info)
  - `--accent` (#22D3EE) on `--bg-base` (#0C1017): ratio 8.5:1
- Cursor colors all have >4.5:1 contrast against `--bg-base`
- Focus indicators use `--accent` border (clearly visible on dark)
- Motion: respect `prefers-reduced-motion` -- disable cursor breathe, card rotation, ghost traces
- Drag and drop: keyboard accessible via @dnd-kit's built-in keyboard support
