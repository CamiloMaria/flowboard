## Typography

4 sizes, 2 weights. Collapsed from DESIGN.md's full type scale to the minimum needed for this phase.

| Role | Font | Size | Weight | Line Height | Usage in this phase |
|------|------|------|--------|-------------|---------------------|
| Display | Space Grotesk | 24px (1.5rem) | 600 | 1.25 | Board name in header |
| Heading | Space Grotesk | 18px (1.125rem) | 600 | 1.25 | Column/list headers, modal title |
| Body | DM Sans | 14px (0.875rem) | 400 | 1.5 | Card titles, card description textarea, form labels, button text, toast title, toast body, UI text, breadcrumb text, input placeholders |
| Meta | JetBrains Mono | 12px (0.75rem) | 400 | 1.5 | Timestamps, card count badges, connection status label, created date in modal, captions |

**Weight policy:** Only 400 (regular) and 600 (semi-bold) are used. Space Grotesk uses 600 exclusively. DM Sans uses 400 exclusively. JetBrains Mono uses 400 exclusively. No 500 or 700 weights are loaded or referenced.

**DESIGN.md mapping:** DESIGN.md declares sizes 12–32px and weights 400–700. This phase contract narrows to 4 sizes (12, 14, 18, 24) and 2 weights (400, 600) for visual hierarchy clarity. The full DESIGN.md scale remains available for future phases.

**Source:** DESIGN.md type scale, constrained for Phase 2

## Interaction Contracts

### Drag and Drop (@dnd-kit/react 0.3.x)

| Interaction | Behavior |
|-------------|----------|
| Drag start | Card scales to 1.03, shadow deepens, cursor: grabbing. Original position shows dashed placeholder. |
| During drag | Overlay follows cursor with 40ms spring lag. +/-2deg rotation based on velocity. |
| Drop zone | 2px accent-colored line appears between cards at insertion target. Target column border highlights. |
| Drop (success) | Card springs into position: spring(stiffness: 120, damping: 14). Other cards shift with layout animation spring(stiffness: 200, damping: 25, mass: 0.8) + 30ms stagger. Optimistic update — no loading state. |
| Drop (failure) | Card animates back to original position with ease-out 300ms. Toast: error type. Other cards revert with same layout spring. |
| Auto-scroll | Board scrolls horizontally when drag cursor enters 60px zone at left/right board edges. Smooth acceleration curve. |

### Remote Updates (Socket.io)

| Event | Visual behavior |
|-------|----------------|
| card:move (remote) | Card animates from old position to new position, spring(stiffness: 200, damping: 25, mass: 0.8). Other cards shift smoothly. |
| card:create (remote) | New card fades in + scales from 0.95, 250ms, ease-out. |
| card:delete (remote) | Card fades out + scales to 0.95, 300ms, ease-in. Gap closes with layout animation. |
| card:update (remote) | Title text updates instantly (no animation). |
| list:create (remote) | New column slides in from right, 300ms, ease-out. |
| list:update (remote) | Name updates instantly. |
| list:delete (remote) | Column fades + scales out, 300ms, ease-in. Remaining columns shift left with layout animation. |

### Optimistic Update Pattern

| Step | Behavior |
|------|----------|
| onMutate | Snapshot current TanStack Query cache. Apply change to cache immediately. UI reflects change instantly. |
| onError | Revert cache to snapshot. Card animates back with ease-out 300ms. Show toast: "{action} failed. Please try again." |
| onSettled | Re-sync from server (invalidateQueries) to ensure consistency. |

### Keyboard Interactions

| Key | Context | Action |
|-----|---------|--------|
| Escape | Modal open | Close modal |
| Escape | Inline edit active | Cancel edit, revert value |
| Enter | Inline edit active | Save edit |
| Enter | Add list input | Create list |
| Enter | Add card input | Create card |

## Accessibility

| Concern | Implementation |
|---------|----------------|
| Contrast | All text meets WCAG AA. `--text-primary` on `--bg-card`: 9.8:1. `--accent` on `--bg-base`: 8.5:1. |
| Keyboard DnD | @dnd-kit provides built-in keyboard support (Space to pick, Arrow keys to move, Space to drop). |
| Focus management | Modal traps focus. Escape closes modal. Focus returns to triggering card. |
| Reduced motion | `prefers-reduced-motion: reduce` — disable card rotation during drag, disable spring physics (use linear transitions), disable layout stagger. Keep functional animations (opacity changes). |
| Screen reader | Cards: `role="button"`, `aria-label="{card title}"`. Lists: `role="region"`, `aria-label="{list name}"`. Modal: `role="dialog"`, `aria-modal="true"`, `aria-labelledby="modal-title"`. Live regions for toast notifications: `role="status"`, `aria-live="polite"`. |
| Icon-only actions | Column options (3-dot menu): `aria-label="List options"`. Modal close (X): `aria-label="Close modal"`. Toast dismiss (X): `aria-label="Dismiss notification"`. Delete card (Trash icon): `aria-label="Delete card"`. |
