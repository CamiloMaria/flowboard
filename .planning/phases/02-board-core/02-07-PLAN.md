```typescript
export function getSocketId(): string | undefined {
  return socket?.id;
```

Add this after the existing `getSocket()` function. Returns the Socket.io client `id` (which matches the server-side `client.id`), or `undefined` if not connected.

   ```typescript
   // Attach socket ID so the server can exclude this client from broadcasts
   ```

<task type="auto">
  <name>Task 2: Fix move failure crash — success-only invalidation + stale-while-revalidate</name>
  <files>apps/web/src/hooks/useBoardMutations.ts, apps/web/src/pages/BoardPage.tsx</files>
  <action>
**Root cause:** Every mutation's `onSettled` calls `invalidateQueries` unconditionally — even on error. When the backend rejects a move, `onError` restores the snapshot (correct), but then `onSettled` fires and triggers a refetch. If the refetch also fails (e.g., the server is temporarily down), TanStack Query puts the query into error state, which causes BoardPage to render the "Failed to load board" error div — wiping the board even though cached data exists.

**Fix 2 files:**

1. **`apps/web/src/hooks/useBoardMutations.ts`** — Change ALL 7 mutations from `onSettled` to `onSuccess` for cache invalidation:

   For each of the 7 mutation hooks (useCreateList, useUpdateList, useDeleteList, useCreateCard, useUpdateCard, useMoveCard, useDeleteCard), replace the `onSettled` callback:

   **Before (in each mutation):**
   ```typescript
   onSettled: () => {
     queryClient.invalidateQueries({ queryKey: ['board', boardId] });
   },
   ```

   **After (in each mutation):**
   ```typescript
   onSuccess: () => {
     queryClient.invalidateQueries({ queryKey: ['board', boardId] });
   },
   ```

   This ensures:
   - On success: refetch to sync server state (replaces temp IDs, gets real positions)
   - On error: `onError` restores the snapshot and NO refetch happens — the board stays stable with the rolled-back optimistic data

   There are exactly 7 occurrences of `onSettled` to replace. Do NOT change any `onError` callbacks — they are correct.

2. **`apps/web/src/pages/BoardPage.tsx`** — Show cached (stale) data when query is in error state but cache exists:

   Currently (line 21):
   ```typescript
   if (error || !board) return <div className="p-6 text-accent-danger">Failed to load board</div>;
   ```

   Change to show the error message as a dismissible banner while still rendering the board if cache data exists:
   ```typescript
   if (!board && (isLoading || error)) {
     return isLoading
       ? <BoardSkeleton />
       : <div className="p-6 text-accent-danger">Failed to load board</div>;
   }
   if (!board) return <div className="p-6 text-accent-danger">Failed to load board</div>;
   ```

   Also remove the separate `if (isLoading)` check above since it's folded into the combined guard. The key insight: when `error` is truthy but `board` still has cached data (TanStack Query preserves stale data), render the board normally instead of the error screen. This makes transient failures invisible to the user.

   The final guard logic should be:
   ```typescript
   if (isLoading && !board) return <BoardSkeleton />;
   if (error && !board) return <div className="p-6 text-accent-danger">Failed to load board</div>;
   if (!board) return <div className="p-6 text-accent-danger">Failed to load board</div>;
   ```

```typescript
function EmptyListPlaceholder({ listId }: { listId: string }) {
```
