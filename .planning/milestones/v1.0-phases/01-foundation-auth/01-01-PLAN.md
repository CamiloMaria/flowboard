       "lint": {},
       "test": {}
     }

   REDIS_URL=redis://localhost:6379

   JWT_REFRESH_SECRET=change-me-refresh-in-production

   - `src/index.ts`: Barrel export all types from the three files

**Per D-01, D-03:** Create both apps and install all Phase 1 dependencies.

**NestJS API (`apps/api/`):**

1. `package.json`: name `@flowboard/api`, scripts: `dev` = `nest start --watch`, `build` = `nest build`, `lint` = `eslint`, `test` = `jest`. Dependencies per RESEARCH.md Installation section:
   - Runtime: `@nestjs/core`, `@nestjs/common`, `@nestjs/platform-express`, `@nestjs/websockets`, `@nestjs/platform-socket.io`, `@nestjs/jwt`, `@nestjs/passport`, `@nestjs/config`, `passport`, `passport-jwt`, `@prisma/client`, `socket.io`, `bcrypt`, `cookie-parser`, `class-validator`, `class-transformer`, `ioredis`, `yjs`, `y-websocket`, `ws`, `rxjs`, `reflect-metadata`, `@flowboard/shared` (workspace:*)
   - DevDependencies: `prisma`, `@types/bcrypt`, `@types/cookie-parser`, `@types/passport-jwt`, `@types/ws`, `@types/express`, `typescript`, `@nestjs/cli`, `@nestjs/schematics`, `@nestjs/testing`, `ts-node`, `tsconfig-paths`, `@types/node`

2. `tsconfig.json`: strict, ES2022 target, experimentalDecorators + emitDecoratorMetadata (NestJS requires both), paths: `@flowboard/shared` → `../../packages/shared/src`

3. `tsconfig.build.json`: extends tsconfig.json, excludes node_modules and test files

4. `nest-cli.json`: `{ "collection": "@nestjs/schematics", "sourceRoot": "src", "compilerOptions": { "tsConfigPath": "tsconfig.build.json" } }`

5. `src/main.ts`: NestJS bootstrap function — create app, `app.setGlobalPrefix('api')`, enable ValidationPipe globally (whitelist: true, forbidNonWhitelisted: true), use cookie-parser middleware, enable CORS for localhost:5173, `await app.listen(process.env.PORT ?? 3001)`. Add a console.log for startup confirmation: `console.log('FlowBoard API running on port ${port}')`.

6. `src/app.module.ts`: Root module importing `ConfigModule.forRoot({ isGlobal: true })`. Registers `AppController`.

7. `src/app.controller.ts`: Single GET `/` endpoint (maps to `/api/` with prefix) returning `{ status: 'ok', timestamp: new Date().toISOString() }`. Decorated with `@Public()` — but since guards aren't set up yet, just use `@Get()` for now.

8. `src/app.service.ts`: Empty AppService for NestJS convention.

**Vite React Frontend (`apps/web/`):**

1. `package.json`: name `@flowboard/web`, scripts: `dev` = `vite`, `build` = `tsc -b && vite build`, `lint` = `eslint`. Dependencies per RESEARCH.md:
   - Runtime: `react`, `react-dom`, `react-router`, `socket.io-client`, `@flowboard/shared` (workspace:*)
   - DevDependencies: `vite`, `@vitejs/plugin-react`, `tailwindcss`, `typescript`, `@types/react`, `@types/react-dom`

2. `vite.config.ts`: Per D-03, configure proxy — `/api` target http://localhost:3001 changeOrigin true, `/socket.io` target http://localhost:3001 ws true, `/yjs` target http://localhost:3001 ws true.

3. `index.html`: Standard Vite React entry with `<div id="root"></div>`, title "FlowBoard"

4. `src/main.tsx`: React 19 `createRoot` rendering `<App />`

5. `src/App.tsx`: Minimal component with `<h1>FlowBoard</h1>` placeholder

6. `src/app.css`: Import tailwindcss: `@import "tailwindcss";` (Tailwind v4 CSS-first per RESEARCH.md gotcha #5)

7. `src/vite-env.d.ts`: Vite client types reference

8. `tsconfig.json`, `tsconfig.app.json`, `tsconfig.node.json`: Standard Vite React TypeScript configs with path alias for `@flowboard/shared`
