    **Create `apps/api/src/demo/cursor-path.ts` — Smooth cursor arc generator (D-02):**
    ```typescript
    export interface CursorPoint { x: number; y: number; }

    /**
     * Generate points along a quadratic Bezier curve for smooth cursor movement.
     * @param start - starting position
     * @param end - target position
     * @param steps - number of intermediate points (default 20 for ~60fps over ~330ms)
     * @returns Array of {x,y} points along the curved path
     */
    export function generateCursorPath(start: CursorPoint, end: CursorPoint, steps = 20): CursorPoint[];
    ```
    - Control point: perpendicular offset from midpoint. Offset = 15-30% of distance, randomly left or right (simulates natural mouse arcs)
    - Quadratic Bezier formula: `B(t) = (1-t)²·P0 + 2(1-t)t·P1 + t²·P2` for t in [0,1] sampled at `steps` intervals
    - Add slight easing: more samples at start (acceleration) and end (deceleration) via easeInOut t-mapping
    - Export a `randomizeDelay(min: number, max: number): number` helper for human-like timing jitter

    **Create `apps/api/src/demo/choreography.ts` — 60-second scripted sequence (D-01, D-04):**

    The choreography function signature:
    ```typescript
    export async function runChoreography(deps: {
      boardService: BoardService;
      boardGateway: BoardGateway;
      presenceService: PresenceService;
      signal: AbortSignal;
    }): Promise<void>;
    ```

    **Choreography design (D-01 — parallel wow-first):**
    Bots must already be visibly active at t=0. The function starts all 3 bots simultaneously:

    Timeline (approximate — exact seconds are agent discretion):
    - **t=0-5s (Wow moment):** All 3 bots active simultaneously.
      - Maria: cursor roams across "In Progress" column cards, pauses on each (reading gesture)
      - Carlos: cursor moves to "To Do" column, hovers over a card
      - Ana: cursor roams board header area, then moves to "Review" column
    - **t=5-15s (Card moves showcase):**
      - Maria: drags a card from "In Progress" to "Review" — broadcast `card:move` event via boardGateway. Cursor follows arc to target column.
      - Carlos: drags a card from "To Do" to "In Progress" — same pattern
      - Ana: cursor watches Maria's move (roams to where card landed), then moves a card from "Backlog" to "To Do"
    - **t=15-30s (Editing showcase):**
      - Carlos: moves cursor to a card in "In Progress", waits 1s (reading), opens card detail modal (frontend handles this via card:update event with description changes)
      - Maria: cursor moves to same card area (converging for D-04 collab moment)
    - **t=30-45s (CRDT climax — D-04):**
      - Carlos: types into card description via Yjs (simulate by calling updateCard with description text additions, broadcast description changes)
      - Maria: types into same card description simultaneously — character-level CRDT sync visible to viewer
      - Ana: continues independent cursor roaming and moves a card from "Review" to "Done"
    - **t=45-60s (Wind down):**
      - Collaborative editing finishes
      - Bots do independent card moves and cursor roaming, slowing down pace
      - At t=60s, transition to random behavior

    **Implementation approach:**
    - Use async/await with `sleep(ms, signal)` helper that rejects on abort
    - Each bot runs as a parallel async function: `Promise.all([mariaBehavior(), carlosBehavior(), anaBehavior()])`
    - Cursor movement: call `boardGateway.broadcastToBoard(DEMO_BOARD_ID, 'presence:cursor', { userId, name, color, x, y, boardId })` for each point in the path with ~16ms intervals (60fps)
    - Card moves: call `boardService.moveCard()` then `boardGateway.broadcastToBoard(DEMO_BOARD_ID, 'card:move', { card, fromListId, toListId })` — no excludeSocketId since bots have no socket
    - Card description editing: call `boardService.updateCard(cardId, { descriptionText: newText })` then broadcast `card:update`. For the CRDT climax, both bots update the same card with interleaved character additions (simulate typing by appending characters every 100-200ms)
    - Read board data via `boardService.getBoard(DEMO_BOARD_ID)` at start to get actual card IDs and positions for targeting

    **Abort safety:** Every `sleep()` call checks `signal.aborted` and throws if true. Promise.all wraps in try/catch to handle abort gracefully.

    **Wire into DemoService:**
    - In `startBots()`, after registering presence, call `runChoreography(deps).then(() => this.startRandomBehavior()).catch(err => { if (err.name !== 'AbortError') console.error(err); })`
    - Add `private startRandomBehavior()` method that calls `runRandomBehaviorLoop` from random-behavior.ts

    ```typescript
    export type BotAction = 'move_card' | 'cursor_roam' | 'idle_pause';

    export interface BotPersonality {
      bot: BotUser;
      weights: Record<BotAction, number>;  // Normalized weights (sum to 1)
    }

    export const BOT_PERSONALITIES: BotPersonality[] = [
      { bot: BOT_USERS[0], weights: { move_card: 0.5, cursor_roam: 0.3, idle_pause: 0.2 } },  // Maria prefers moves
      { bot: BOT_USERS[1], weights: { move_card: 0.2, cursor_roam: 0.5, idle_pause: 0.3 } },  // Carlos prefers cursor/typing
      { bot: BOT_USERS[2], weights: { move_card: 0.3, cursor_roam: 0.5, idle_pause: 0.2 } },  // Ana prefers roaming
    ];
    ```

    Per D-03: one bot action every 5-10 seconds (randomized). Feels like a calm team in flow state.
    Per D-04: random phase does NOT open cards for editing — only moves and cursor roaming.

    `export function weightedRandom(weights: Record<string, number>): string` — standard weighted random selection.

    `export async function runRandomBehaviorLoop(deps: ChoreographyDeps): Promise<void>`:
    - Infinite loop (until abort)
    - Each iteration: pick a random bot, pick a weighted-random action for that bot
    - Execute action:
      - `move_card`: pick a random card, move to a random different list. Cursor arc to card → pause 500ms → arc to target list → broadast card:move
      - `cursor_roam`: generate a Bezier path to a random board position, emit cursor events along path
      - `idle_pause`: do nothing for the interval (natural break)
    - Sleep 5000-10000ms (randomized) between actions
    - Wrap in try/catch for AbortError
