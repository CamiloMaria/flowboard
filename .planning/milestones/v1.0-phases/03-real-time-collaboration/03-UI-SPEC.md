## Spacing Scale

Declared values (multiples of 4, per DESIGN.md `--space-*` tokens):

| Token | Value | Usage in Phase 3 |
|-------|-------|-------------------|
| xs | 4px | Cursor name pill vertical padding, avatar overlap offset, inline code vertical padding, floating toolbar button gap |
| sm | 8px | Avatar group gap, editor toolbar gaps, cursor label offset, cursor name pill horizontal padding |
| md | 16px | Editor section padding, modal section gaps |
| lg | 24px | Modal outer padding (existing), board outer padding (existing) |
| xl | 32px | Not used in Phase 3 |
| 2xl | 48px | Not used in Phase 3 |
| 3xl | 64px | Not used in Phase 3 |

Exceptions:
- Avatar overlap: -8px negative margin-left (DESIGN.md avatar group spec)
- Cursor name pill offset: 6px below-right of arrow tip (custom position, not grid-aligned)
- Touch target for avatar group container: min 32px height
- **2px micro-padding:** Used for cursor name pill vertical padding (`2px 8px`), TipTap inline code padding (`2px 4px`), and floating toolbar button gap (`2px`). Justification: these are inline or micro-elements rendered at 12px font size where the minimum grid unit of 4px creates excessive visual padding relative to the text — 4px vertical padding on a 12px-tall pill inflates height by 67% vs. 33% with 2px. The 2px value is reserved exclusively for sub-grid-unit inline contexts.

Source: DESIGN.md spacing system, CONTEXT.md D-09.

## Typography

| Role | Font | Size | Weight | Line Height | Phase 3 Usage |
|------|------|------|--------|-------------|---------------|
| Body | DM Sans | 14px (0.875rem) | 400 | 1.5 | TipTap editor body text, cursor name labels, avatar initials text (OnlineUsers 32px and CoEditorAvatars 24px), TipTap inline code (JetBrains Mono at 14px) |
| Label | DM Sans | 12px (0.75rem) | 400 | 1.25 | Cursor name pill text, co-editor avatar labels, "Reconnecting..." banner text, TipTap in-editor cursor name labels, overflow "+N" pill text, connection status label (JetBrains Mono at 12px) |
| Heading | Space Grotesk | 18px (1.125rem) | 600 | 1.25 | Modal title (existing), "Description" section label |

**Font weights used in Phase 3:** 400 (regular) and 600 (semibold). Two weights only.
- 400: All body text, labels, captions, editor content, cursor pills
- 600: Section headings, modal titles, emphasis

**TipTap editor-specific typography:**
- Paragraph: DM Sans 400, 14px, line-height 1.5, color `--text-primary` (#E2E8F0)
- Bold: DM Sans 600 (weight bump only, no size change)
- Inline code: JetBrains Mono 400, 14px, bg `--bg-card` (#1A2233), border-radius 4px, padding 2px 4px
- Placeholder: DM Sans 400, 14px, color `--text-muted` (#64748B)
- Cursor name label inside editor: DM Sans 400, 12px, white text on user-color background

**Rationale for collapses (revision):**
- 10px → 12px: Cursor name labels inside TipTap editor now use 12px (Label role) instead of 10px. 12px is the minimum legible size and matches the board-level cursor name pill.
- 13px → 14px: Avatar initials text now uses 14px (Body role) instead of 13px. Visually negligible difference at avatar scale, maintains grid alignment.
- 500 → dropped: TipTap bold uses 600 instead of 500 for stronger visual hierarchy. Avatar initials use 400 (weight distinction unnecessary — color is the primary differentiator).

Source: DESIGN.md typography scale, CONTEXT.md D-01/D-04.

## Component Specifications

### RemoteCursor

**Visual:** Figma-style multiplayer cursor — filled arrow triangle (16×20px) in user's color + rounded pill label below-right showing user name.

```
     ▲          ← 16×20px filled triangle, fill: user color
    ╱ ╲
   ╱   ╲
  ╱─────╲
      ┌──────────┐
      │  Maria   │  ← pill: user color bg, white text, DM Sans 12px, radius-full
      └──────────┘
```

| Property | Value |
|----------|-------|
| Arrow size | 16px wide × 20px tall SVG path |
| Arrow fill | User's assigned color (solid) |
| Arrow stroke | none |
| Name pill font | DM Sans 400, 12px (--text-xs) |
| Name pill text color | `#FFFFFF` (white on colored background) |
| Name pill background | User's assigned color |
| Name pill padding | 2px 8px (2px exception — see Spacing Scale) |
| Name pill border-radius | 9999px (--radius-full) |
| Name pill offset | 6px right and 2px down from arrow tip |
| Glow effect | `filter: drop-shadow(0 0 8px {userColor})` |
| z-index | 40 (below modals at 50, above cards) |
| pointer-events | none (cursors don't block board interaction) |

**States:**
- **Moving:** Glow at static 8px spread. Position interpolated at 60fps with 60ms lerp smoothing.
- **Idle (3+ seconds stationary):** Glow breathe animation — `drop-shadow` oscillates between 8px and 14px spread over a 3-second sine cycle. CSS `@keyframes cursor-breathe` already defined in DESIGN.md.
- **Exiting:** 400ms fade to opacity 0 (`ease-in`). Glow lingers for an additional 600ms then dissolves (ghost trace effect). Total exit: 1000ms.
- **Entering:** Fade in from 0 to 1 over 400ms (`ease-out`), slide from nearest screen edge to cursor position.

**Reduced motion:** When `prefers-reduced-motion` is active, disable breathe animation and ghost trace. Cursor still renders and moves, but without glow oscillation. Exit is instant fade (200ms).

Source: DESIGN.md cursor section, CONTEXT.md D-05/D-06/D-07/D-08, REQUIREMENTS.md PRES-03/PRES-04.

### OnlineUsers (Avatar Strip)

**Visual:** Horizontal stack of colored initial circle avatars with slight overlap. Max 5 visible, "+N" overflow pill for additional users.

```
[M][C][A][You] +2
```

| Property | Value |
|----------|-------|
| Avatar diameter | 32px |
| Avatar shape | circle (border-radius: 9999px) |
| Avatar fill | User's assigned color (solid) |
| Avatar text | Single uppercase initial (first letter of name), or two initials for two-word names |
| Avatar text font | DM Sans 400, 14px |
| Avatar text color | `#0C1017` (--bg-base) for light-on-dark readability on bright user colors |
| Avatar border | 2px solid `#0C1017` (--bg-base) — creates visual separation in stack |
| Avatar overlap | -8px margin-left (first avatar has no negative margin) |
| Max visible | 5 avatars |
| Overflow pill | "+N" text, `#64748B` (--text-muted) color, DM Sans 400 12px, bg `#141B27` (--bg-surface), 32px height, padding 0 8px, border-radius 9999px, border 2px solid `#0C1017` |
| Container | flex row, items-center, no background (sits directly in header) |

**Animations:**
- **Join:** Scale from 0 → 1.0 + fade in, spring easing (stiffness 300, damping 20), 250ms. New avatar appears at end of stack, other avatars shift smoothly via `motion` layout animation.
- **Leave:** Scale from 1.0 → 0 + fade out, 200ms ease-in. Stack re-adjusts via layout animation.
- **Reduced motion:** No scale animation, instant appear/disappear.

**Layout decision (D-12 — agent discretion):** Place `<OnlineUsers>` to the LEFT of `<ConnectionStatus>` in the board header. Keep `<ConnectionStatus>` visible — it serves a different function (transport health) from avatar presence (who's here). The avatars being visible IS the primary connection indicator for recruiters, but the explicit status dot helps developers debug.

Source: DESIGN.md presence section, CONTEXT.md D-09/D-10/D-11/D-12.

### CursorOverlay

**Visual:** Absolute-positioned transparent layer covering the entire board canvas. Renders all remote cursors. Does not intercept pointer events.

| Property | Value |
|----------|-------|
| Position | `absolute`, `inset-0` (covers entire board canvas) |
| pointer-events | `none` |
| z-index | 40 |
| Overflow | hidden (cursors clip at board edges) |
| Children | `<RemoteCursor>` for each online user (excluding self) |

**Cursor position mapping:** Cursor x/y coordinates are relative to the board canvas element. When the board scrolls horizontally, cursor positions must account for `scrollLeft` offset. ## Interaction Contracts

### Cursor Broadcasting

| Property | Value |
|----------|-------|
| Emit event | `presence:cursor` via Socket.io |
| Emit throttle | 50ms on client-side (D-21) |
| Payload | `{ x: number, y: number, boardId: string }` |
| Coordinates | Relative to board canvas scroll container |
| Interpolation | 60fps linear interpolation with 60ms lerp on receiving client |
| Idle detection | 3 seconds of no movement triggers breathe animation on remote clients |

### Yjs Provider Lifecycle

| Trigger | Action |
|---------|--------|
| Card modal opens | Create `WebsocketProvider('ws://.../yjs', 'card:{cardId}', ydoc)` with JWT token in URL params |
| Card modal closes | `provider.destroy()`, `ydoc.destroy()` — clean resource teardown |
| Provider disconnects | Show `<ReconnectBanner>` in "reconnecting" state. User keeps typing (Yjs buffers locally) |
| Provider reconnects | Hide banner. Yjs auto-merges divergent edits |
| Reconnect fails (6 attempts) | Show "Connection lost" banner with "Copy content" button |

Source: CONTEXT.md D-13/D-16, ARCHITECTURE.md Flow 2.

### Presence Join/Leave

| Trigger | Client Action | Visual Effect |
|---------|---------------|---------------|
| `presence:join` received | Add user to Zustand `onlineUsers` array | Avatar scales in (spring), cursor fades in from edge |
| `presence:leave` received | Remove user from `onlineUsers`, mark cursor as exiting | Avatar scales out, cursor ghost-fades (400ms + 600ms glow) |
| Own socket connects | Emit `board:join` (existing). Server broadcasts `presence:join` to room | Own avatar does NOT appear in strip (only remote users shown) |
| Own socket disconnects | Server emits `presence:leave` to room | Remote clients see avatar leave |

Source: CONTEXT.md D-22, ARCHITECTURE.md Flow 3.

## Animation Specifications (Phase 3 Specific)

| Animation | Duration | Easing | Implementation |
|-----------|----------|--------|----------------|
| Remote cursor move | Continuous | Linear interpolation, 60ms lerp | `requestAnimationFrame` loop interpolating between last known and current position |
| Cursor idle breathe | 3s cycle | Sine (`ease-in-out`) | CSS `@keyframes cursor-breathe` — glow oscillates 8px → 14px → 8px |
| Cursor enter | 400ms | ease-out | `motion.div` initial={{ opacity: 0 }} animate={{ opacity: 1 }} |
| Cursor exit | 400ms fade + 600ms glow linger | ease-in | Two-phase: opacity 0 over 400ms, then glow-only element fades over 600ms |
| Avatar join | 250ms | spring(300, 20) | `motion.div` initial={{ scale: 0, opacity: 0 }} animate={{ scale: 1, opacity: 1 }} |
| Avatar leave | 200ms | ease-in | `motion.div` exit={{ scale: 0, opacity: 0 }} |
| Avatar stack reflow | 200ms | spring(200, 25) | `motion.div` layout prop on each avatar |
| Floating toolbar appear | 150ms | ease-out | Fade + 4px translateY |
| Floating toolbar disappear | 100ms | ease-in | Fade |
| Reconnect banner show | 200ms | ease-out | Height 0 → 32px + fade in |
| Reconnect banner hide | 200ms | ease-in | Height 32px → 0 + fade out |
| Co-editor avatar appear | 200ms | ease-out | Scale 0.8 → 1 + fade in |
| Co-editor avatar disappear | 150ms | ease-in | Scale 1 → 0.8 + fade out |

## Accessibility

|---------|----------------|
