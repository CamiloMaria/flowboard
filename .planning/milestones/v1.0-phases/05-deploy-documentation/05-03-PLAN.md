    **Per D-16:** Create `apps/web/vercel.json` following NotifyHub pattern:

    ```json
    {
      "rewrites": [
        { "source": "/(.*)", "destination": "/index.html" }
      ],
      "headers": [
        {
          "source": "/assets/(.*)",
          "headers": [
            { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
          ]
        }
      ]
    }
    ```

    This handles SPA client-side routing (all paths serve index.html) and sets immutable cache headers on hashed assets (Vite puts built JS/CSS in `/assets/`).

    **Per D-17:** Update `apps/web/vite.config.ts` to read `VITE_API_URL` for production:

    The frontend code currently uses relative paths for API calls (e.g., `/api/boards`), Socket.io (connects to `/socket.io/`), and y-websocket (connects to `/yjs/`). In production, the frontend is on Vercel but the API is on Oracle Cloud — different domains.

    The Vite proxy handles this in local dev. For production, the frontend code needs to use the full API URL. This is done by:

    1. Checking if any existing API/socket client code references base URLs that need to be configurable. Look for the api helper in `apps/web/src/lib/api.ts` and socket client in `apps/web/src/lib/socket.ts` — these likely use relative paths that need `VITE_API_URL` prefix in production.

    2. Do NOT modify the Vite proxy config — it works perfectly for local dev. Only ensure `VITE_API_URL` is available as an env var that the frontend code can read via `import.meta.env.VITE_API_URL`.

    3. The actual code changes to prepend `VITE_API_URL` to fetch calls and socket connections will be handled by the executor reading the existing api.ts/socket.ts files and updating the base URL logic. The pattern:
       ```typescript
       const API_BASE = import.meta.env.VITE_API_URL || '';
       // Then: fetch(`${API_BASE}/api/boards`)
       // Socket.io: io(API_BASE || window.location.origin, { path: '/socket.io/' })
       // y-websocket: new WebsocketProvider(`${wsBase}/yjs`, ...)
       ```

    4. In Vercel dashboard, the user sets `VITE_API_URL=https://their-oracle-api-domain.com` as an environment variable. In local dev, it's unset so defaults to empty string (relative paths, proxied by Vite).

    **Vercel project settings (documented for user):**
    - Root Directory: `apps/web`
    - Build Command: `cd ../.. && pnpm install && pnpm --filter @flowboard/web build` (or let Vercel auto-detect)
    - Output Directory: `dist`
    - Environment Variable: `VITE_API_URL` = `https://your-api-domain.com`
